- 判別可能ユニオンは、共通のプロパティにリテラル型を持たせるのが基本
```ts
type Result<T, Error> = 
    { success: true, value: T },
    | { success: false, error: E }
```

- boolean でも作れるが、状態が増えるなら文字列リテラルのほうが扱いやすい
```ts
type Result<T, Error> = 
    { kind: "ok", value: T },
    | { kind: "error", error: E }
```

- ジェネリクスの型引数名は、組み込み型と紛らわしくない名前にする
```ts
type Result<T, E> = 
    { kind: "ok", value: T },
    | { kind: "error", error: E }
```

- 制約は必要なときだけ extends で付ける
```ts
type Result<T, E extends Error> = 
    { kind: "ok", value: T },
    | { kind: "error", error: E }
```

- Result のような戻り値は readonly にすると安全
```ts
type Result<T, E extends Error = Error> =
  | { readonly kind: "ok"; readonly value: T }
  | { readonly kind: "err"; readonly error: E };
```
